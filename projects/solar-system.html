Project 02 — 3D Modeling & Animation

3D Solar System

A fully modeled and animated solar system built in Blender 4.2, featuring every planet, the Sun, and Pluto — with orbital motion, textures, and Python scripting.

Blender 4.2 Python 3D Modeling Cycles Render PBR Textures

Section 01

About the Project

This project is a 3D simulation of our solar system created entirely within Blender 4.2. The goal was to model each celestial body with accurate relative sizing, apply realistic PBR texture materials, and script the orbital motion programmatically using Python.

The project demonstrates the integration of artistic 3D modeling with technical scripting, producing a rendered animation that captures the scale and motion of the solar system. Submitted as a major output for the subject Multimedia and Authoring.

Project Details

Tools Used

  • Blender 4.2 — primary 3D modeling and animation software
  • Python 3.11 — scripting orbital mechanics and automation
  • NASA texture maps — surface material references
  • Cycles renderer — photorealistic rendering engine

Render Info

  • Renderer: Cycles (GPU accelerated)
  • Resolution: 1920 × 1080 (1080p)
  • Frame rate: 24 FPS
  • Samples: 128 per frame
  • Output format: MP4 (H.264)

Key Features

  • All 8 planets + Pluto + Sun modeled and textured
  • Orbital motion automated via Python script
  • Axial tilt applied to each planet
  • Sun emission shader with volumetric glow
  • Saturn ring system with transparency texture
  • HDRI space environment background

Texture Materials

  • Sun — emission shader, noise-based surface texture
  • Earth — diffuse map, cloud layer, specular water map
  • Gas giants — procedural band textures (Jupiter, Saturn)
  • Ice giants — tinted emission + diffuse blend (Uranus, Neptune)
  • Rocky planets — bump-mapped crater surface textures
  • Saturn rings — alpha-mapped ring plane geometry

Python Script

Code

Blender Python Script

The following script automates orbital motion by keyframing each planet's location based on its orbital period and distance from the Sun. 

# solar_system.py — Blender Python Orbital Animation Script
# Multimedia and Authoring — TUP Taguig

import bpy
import math

# Planet data: (name, distance_au, orbital_period_days, radius)
PLANETS = [
    ("Mercury", 0.39, 87.97,  0.38),
    ("Venus",   0.72, 224.70, 0.95),
    ("Earth",   1.00, 365.25, 1.00),
    ("Mars",    1.52, 686.97, 0.53),
    ("Jupiter", 5.20, 4332.59,11.21),
    ("Saturn",  9.58, 10759.22,9.45),
    ("Uranus",  19.2, 30688.5, 4.01),
    ("Neptune", 30.1, 60182.0, 3.88),
    ("Pluto",   39.5, 90560.0, 0.19),
]

# Scale factors for Blender scene units
AU_SCALE     = 5.0   # 1 AU = 5 Blender units
RADIUS_SCALE = 0.4   # planet radius multiplier
FRAME_SCALE  = 0.05  # days per frame (compress time)
TOTAL_FRAMES = 960   # 40 seconds at 24 fps


def create_planet(name, distance, radius):
    """Create a UV sphere for a planet at the given distance."""
    bpy.ops.mesh.primitive_uv_sphere_add(
        radius=radius * RADIUS_SCALE,
        location=(distance * AU_SCALE, 0, 0)
    )
    obj = bpy.context.object
    obj.name = name
    return obj


def animate_orbit(obj, distance, period_days, total_frames):
    """Keyframe circular orbit around the origin."""
    r = distance * AU_SCALE
    for frame in range(0, total_frames + 1):
        angle = (2 * math.pi) * (frame * FRAME_SCALE) / period_days
        x = r * math.cos(angle)
        y = r * math.sin(angle)
        obj.location = (x, y, 0)
        obj.keyframe_insert(data_path="location", frame=frame)
    # Set linear interpolation for smooth orbits
    for fcurve in obj.animation_data.action.fcurves:
        for kp in fcurve.keyframe_points:
            kp.interpolation = 'LINEAR'


# ── Main execution ─────────────────────────────────────────
for name, dist, period, radius in PLANETS:
    planet_obj = create_planet(name, dist, radius)
    animate_orbit(planet_obj, dist, period, TOTAL_FRAMES)
    print(f"Animated: {name}")

print("Solar system animation complete.")
